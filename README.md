# Processo e Sviluppo del Software
Questa repository contiene i tre assignment che sono stati svolti durante il corso di Processo e Sviluppo del Software A.A 25/26

### Assignment 1: Requirements Elicitation 
Analisi del sistema As-Is e definizione del sistema To-Be. Studio del background, interviste agli stakeholder, questionari utenti e definizione dei requisiti funzionali/non funzionali per l'integrazione di servizi di prenotazione.

### Assignment 2: CI/CD Pipeline 
Progettazione e implementazione di una pipeline di automazione (GitLab CI/CD) strutturata in 6 stage: build, verify, test, package, release e docs.

### Assignment 3: Risk Analysis 
Identificazione e valutazione dei rischi software tramite checklist. Definizione della Top 10 dei rischi e pianificazione delle strategie di mitigazione (Management) e di emergenza (Contingency).

Voto finale: 30 e lode
